git学习总结

git的前世今生

​ git的来源是Linux之父，传说是其在开源社区Linux的版本控制工具商撤销合同之后，在两周内完成了现在git的雏形，斯国一~

​ 现在说下版本控制工具的发展：

​ 首先是最开始的本地控制，就相当于现在的项目的版本文档，当然会出现管理混乱的情况（这个是我一直想吐槽的，开发文档管理很乱，开发或测试回头去看可能自己都不知道自己修改了啥）。

​ 随后出现了SVN（集中管理），其有中心服务器可以统一管理所有版本，但是万一哪天中心服务器挂了，computerA和computerB之间就没法进行版本交互了，因为其本地不存在版本历史记录。

​ 最后出现了git（分布式管理），不光在远程仓库保存了所有历史记录，而且在computerA和computerB的本地也保存了历史记录，就不存在版本无法交互的情况，而且本地也可以查看历史版本记录。

git常用命令

初始化：

  1. git init：初始化git
  2. echo “init” >> READMEN.md
  3. git add README.md
  4. git commit -m “init”
  5. git add remote origin git@xxx.git
  6. git push -u origin master

过程中：

  1. git stash
  2. git pull –rebase
  3. git stash pop
  4. git push

分支：

  1. git status
  2. git checkout xx
  3. git checkout -b newbranch
